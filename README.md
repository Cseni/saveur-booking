# SAVEUR — форма бронирования столика

Тестовое задание на позицию Junior Frontend Developer. Проект реализован на Next.js 14, React, TypeScript, React Hook Form и SCSS Modules.

## Ссылки

- Демо: https://saveur-booking-iota.vercel.app
- Репозиторий: https://github.com/Cseni/saveur-booking

## Функциональность

- обязательные поля: имя, телефон, дата, время и количество гостей;
- валидация после `blur` и при отправке формы;
- нормализация российского номера телефона;
- выбор даты от текущего дня до +90 дней;
- временные слоты с 12:00 до 22:00;
- имитация отправки в течение 1,5 секунды и loader на кнопке;
- экран подтверждения и возврат к новой форме;
- адаптивная вёрстка для мобильных и десктопных экранов;
- unit-тесты функций валидации.

## Локальный запуск

Нужен Node.js 20+.

```bash
npm install
npm run dev
```

После запуска откройте `http://localhost:3000`.

Проверки перед отправкой:

```bash
npm run lint
npm run test
npm run build
npm run format:check
```

## Архитектурные решения

Форма и экран подтверждения разделены на самостоятельные компоненты, а переключение состояний находится в контейнере `BookingPage`. Вся предметная валидация вынесена в `src/utils/validation.ts`, поэтому её можно переиспользовать и тестировать отдельно от интерфейса. Для формы использован React Hook Form с режимом `onBlur`, чтобы ошибки появлялись в соответствии с заданием, а повторная проверка происходила при изменении значения. Даты разбираются как локальные календарные значения, чтобы избежать сдвигов из-за часовых поясов. Стили изолированы через SCSS Modules, а базовые дизайн-токены находятся в `globals.scss`.
